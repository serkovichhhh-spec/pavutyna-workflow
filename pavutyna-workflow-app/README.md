# ПАВУТИНА ВОРКФЛОУ

Локальний робочий додаток для маркетингової операційної моделі Pavutyna.

## Що всередині

- Miro-style карта екосистеми маркетинг-відділу.
- Окремий Trello-style задачник з колонками Backlog, To Do, In Progress, Review, Ready, Done.
- Вхідні задачі природною мовою: додаток визначає відповідального, пріоритет, мітку, дедлайн і генерує чеклист.
- Інспектор картки: редагування назви, опису, статусу, відповідального, дедлайну і чеклисту.
- Фільтри за відповідальним, пріоритетом і текстом.
- Пульс команди: кількість задач, ризики, статуси і навантаження.
- Локальне збереження в браузері, експорт та імпорт JSON.

## Локальний запуск

Відкрий файл:

`E:\Codex\Pavutyna\Designs\outputs\pavutyna-workflow-app\index.html`

Або опублікуй папку як статичний сайт.

## Публікація на GitHub Pages

1. Створи репозиторій `pavutyna-workflow`.
2. Завантаж у нього всі файли з цієї папки.
3. Переконайся, що основна гілка називається `main`.
4. GitHub Actions workflow `Deploy GitHub Pages` опублікує сайт автоматично.
5. Після першого деплою посилання буде доступне у вкладці `Actions` або `Settings -> Pages`.

## Як додавати задачі через Codex

Пиши задачі у чаті у форматі на кшталт:

`Наталі: зробити 3 SMM-пости для GPON у ЖК Варшавський, дедлайн до п'ятниці, пріоритет high`

Я можу внести їх у додаток як готові картки з описом, відповідальним, статусом і чеклистом.

## Спільне сховище

Поточна версія працює локально через `localStorage`. Наступний етап - синхронізація задач із Google Sheets. План структури таблиці описаний у `GOOGLE_SHEETS_ROADMAP.md`.
